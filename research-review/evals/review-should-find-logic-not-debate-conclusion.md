# Eval: 复核关注论证链，而不是直接争论最终观点

## Input

我不同意这份光伏研报的结论，你帮我审查一下它的逻辑有没有问题。

## Must Do

- 重点检查定义、数据、时间口径、推导链和证据权重
- 把“结论我是否同意”与“逻辑是否成立”分开
- 如果逻辑成立但证据薄弱，应如实指出而不是直接反驳结论
- 给出可操作的修正方向

## Must Not

- 直接按自己的观点改写结论
- 把审查变成站队式辩论
- 只评价观点对错，不评价论证是否充分

## Notes

- 允许最终认为结论站不住脚，但必须从论证链切入
- 重点验证 research-review 的角色边界
