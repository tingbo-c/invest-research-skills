# Eval: 估值与投资结论只能在用户明确要求时扩展

## Input

从基本面和估值两个维度看一下宁德时代，现在值不值得关注？

## Must Do

- 先给出基本面判断，再衔接估值或投资含义
- 明确区分“基本面质量”和“当前估值是否匹配”
- 如果输出投资含义，应说明结论成立所依赖的关键条件
- 避免把估值判断写成无条件的买卖口号

## Must Not

- 跳过基本面，直接给投资建议
- 把“值得关注”偷换成确定性的收益承诺
- 只报估值倍数，不解释与基本面的匹配关系

## Notes

- 允许使用条件句、区间判断或情景判断
- 重点验证“显式要求时可以扩展，但扩展要有约束”
