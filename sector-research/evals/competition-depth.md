# Eval: 竞争格局分析不应止步于市占率，需解释格局成因

## Input

帮我分析一下中国方便面行业的竞争格局

## Context

本 eval 测试的核心是"市占率是必要条件，不是终点"。用户提问竞争格局时，自然期待看到市占率数据——但市占率只是格局的表象，不是分析本身。方便面行业格局清晰（康师傅/统一/日清/白象头部集中），数字容易查到，因此很容易让模型在列完玩家和份额后就停下来。真正有价值的分析是：康师傅为什么能长期维持领先？壁垒来自渠道还是规模还是品牌？白象近几年靠什么逆袭，这股力量是否可持续？这是测试模型"有没有往深走"的典型场景。

## Expected Behavior

- 给出主要玩家的市占率或市场位置（基本功，不能缺）
- 在此基础上解释格局成因：头部为什么能长期维持，壁垒来自哪里（渠道覆盖、规模效应、品牌心智）
- 分析格局是否稳定，或有什么力量正在改变它（如白象的国产情感牌、行业整体高端化趋势、方便面消费结构迁移）
- 识别利润主要被哪个环节截留（品牌端还是渠道端），以及对格局稳定性的含义

## Should Not

- 不应止步于玩家列表和市占率数字，不解释背后的结构性逻辑
- 不应把"竞争激烈"或"格局稳定"直接当作分析结论，没有支撑依据
- 不应遗漏对格局变化驱动力的判断，只描述当前现状而不分析动态演变

## 测试提示

运行时在 Input 末尾加"300字以内，覆盖关键点即可"，可将单次测试 token 消耗压缩至约 6-10K。字数有限时模型的优先级选择会直接暴露行为——只会堆数字还是会往深走，一眼可辨。200字太紧，护城河分析容易被字数压掉。
