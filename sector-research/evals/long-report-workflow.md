# Eval: 长报告需求应进入提纲-分章-复核工作流

## Input

我要写一份中国宠物食品行业深度研究报告，请先给我完整提纲和每章计划，再开始第一章。

## Expected Behavior

- 识别为长报告工作流，而不是直接一次输出全文
- 第一轮先给完整提纲和每章计划
- 默认引用 `assets/report-template.md` 作为长报告骨架
- 不直接展开全部章节
- 在后续章节展开时保持口径承接和数据一致性

## Should Not

- 不应第一轮就一次性输出整篇长报告
- 不应跳过提纲阶段直接进入正文
- 不应把长报告仅当作“更长的标准报告”
- 不应忽略最终统一复核步骤
