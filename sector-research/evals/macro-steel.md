# Eval: 钢铁行业应优先触发宏观子分析

## Input

（假设用户已回答完启动前问题后）帮我研究中国钢铁行业，重点看需求拐点、供给格局和利润变化。

## Expected Behavior

- 在正式分析前，先判断钢铁行业是否必须衔接宏观
- 明确指出钢铁是强周期行业，至少应考虑：
  - 地产销售和地产投资
  - 基建与财政发力
  - 制造业投资和信用环境
  - 供给侧约束与原材料成本
- 输出中应说明宏观变量通过需求、开工率、库存、价格和吨钢利润传导到行业
- 可以视情况补充外部环境，但宏观子分析应是主路径
- 最终结论不能只写“行业竞争格局改善”而不交代需求侧驱动

## Should Not

- 不应只做产能、集中度、龙头份额分析而跳过地产/基建/信用周期
- 不应把短期价格波动直接当成行业趋势，而不解释宏观背景
- 不应在没有理由的情况下强行展开完整国际局势分析
- 不应遗漏高频验证信号，例如螺纹钢价格、库存、开工率、地产销售等

## 测试提示

这是“应触发宏观，但不一定每次都触发外部环境”的测试样例。重点看模型会不会先走 `第二点五步`，并把地产、基建、信用作为核心驱动链条写出来。
