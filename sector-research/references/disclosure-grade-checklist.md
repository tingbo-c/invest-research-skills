# 公开披露级数据核查清单

仅在以下场景加载：

- 招股书
- 募资材料
- 融资路演正式材料
- 审计/法律/尽调底稿
- 对外公开发布且需要高可信度背书的研究材料

普通研究、内部讨论、一般投资判断默认不加载本文件，以节约 token。

---

## 硬规则

1. 关键结论优先使用 P1、P2 数据。
2. 没有官方行业总量数据时，可以合理估算，但最底层锚点必须优先来自官方统计、监管披露、行业协会或公司原始披露。
3. P5 数据不得进入核心规模测算主链路。
4. 每个核心数字都必须能回到原始出处。
5. 别人应能按同口径复算出相同或接近结果。

---

## 快速检查表

- [ ] 本次任务是否属于公开披露或高可信度场景？
- [ ] 每个核心结论是否都有 P1 或 P2 级主证据？
- [ ] 若行业总量来自估算，最底层锚点是否来自官方/监管/协会/原始披露？
- [ ] 需求侧推导和供给侧校验是否同时存在？
- [ ] 公式、口径、分层过程、假设是否完整保留？
- [ ] 关键数字是否能追溯到原始发布页面、公告、年报或统计口径？
- [ ] 是否误把招聘、舆情、搜索热度、地图热力、App 活跃当成主证据？
- [ ] 是否给出了误差来源、区间值或情景假设？

---

## 一句话原则

可以推导，但不能漂；可以估算，但源头必须硬。
